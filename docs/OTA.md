## 1. 概述

OTA (Over-The-Air) 升级允许通过无线网络远程更新睡眠舱设备固件。

### 1.1 支持的升级方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| WiFi 升级 | 通过局域网下载固件 | 设备已连接 WiFi |
| BLE 升级 | 通过蓝牙传输固件 | 设备已连接蓝牙 |

### 1.2 升级前提条件

- 设备电量 ≥ 20%
- 网络稳定（WiFi 建议 10MB/s 以上）
- 升级过程中不断电

## 2. 升级流程

### 2.1 完整流程

```
发起升级 → 检查版本 → 下载固件 → 传输固件 → 安装固件 → 重启设备 → 验证版本
```

### 2.2 步骤详解

#### 步骤 1: 检查版本

应用调用 API 检查是否有新版本:

```
GET /devices/:deviceId/ota/check?current_version=1.4.0
```

**响应:**

```json
{
  "success": true,
  "data": {
    "has_update": true,
    "latest_version": "1.5.0",
    "release_notes": "修复蓝牙连接问题，优化睡眠报告",
    "file_size": 2048576,
    "md5": "a1b2c3d4e5f6...",
    "forced": false
  }
}
```

#### 步骤 2: 下载固件

应用下载固件文件:

```
GET /ota/firmware/v1.5.0.bin
```

#### 步骤 3: 传输固件

通过 BLE 将固件数据传输到设备（每包 512 字节）:

```
AA 55 30 [包序号高] [包序号低] [数据块] CRC
```

#### 步骤 4: 安装固件

设备接收完所有数据后开始安装:

```
AA 55 31 [固件版本] CRC
```

#### 步骤 5: 重启验证

设备自动重启并验证新固件:

```
AA 55 32 CRC
```

## 3. 固件版本

### 3.1 版本格式

```
主版本.次版本.修订版本  (如: 1.5.0)
```

| 字段 | 说明 | 升级规则 |
|------|------|----------|
| 主版本 | 重大架构变更 | 强制升级 |
| 次版本 | 功能新增 | 建议升级 |
| 修订版本 | Bug 修复 | 可选升级 |

### 3.2 历史版本

| 版本 | 日期 | 更新内容 |
|------|------|----------|
| 1.5.0 | 2024-03-15 | 新增语音控制，优化蓝牙连接 |
| 1.4.0 | 2024-03-01 | 新增气味控制，修复已知问题 |
| 1.3.0 | 2024-02-15 | 优化睡眠监测算法 |
| 1.2.0 | 2024-01-20 | 新增按摩控制功能 |
| 1.1.0 | 2024-01-05 | 新增灯光控制 |
| 1.0.0 | 2023-12-15 | 初始版本 |

## 4. 升级检查 API

### 4.1 检查更新

```
GET /api/v1/devices/{deviceId}/ota/check
```

**请求参数:**

| 参数 | 类型 | 说明 |
|------|------|------|
| current_version | string | 当前固件版本 |

### 4.2 下载固件

```
GET /api/v1/ota/firmware/{version}
```

## 5. 断点续传

### 5.1 机制

下载中断时，支持从断点继续下载。

### 5.2 实现

使用 HTTP Range 请求:

```
GET /api/v1/ota/firmware/1.5.0.bin
Range: bytes=1024000-
```

## 6. 故障处理

### 6.1 常见错误

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| E101 | 下载失败 | 检查网络，重试 |
| E102 | 固件损坏 | 重新下载 |
| E103 | 空间不足 | 清理设备存储 |
| E104 | 电量不足 | 充电后重试 |
| E105 | 写入失败 | 重试，若失败回滚老版本 |
| E106 | 验证失败 | 自动回滚 |

### 6.2 升级状态查询

```
GET /api/v1/devices/{deviceId}/ota/status
```

**响应:**

```json
{
  "success": true,
  "data": {
    "status": "downloading",
    "progress": 45,
    "downloaded_bytes": 921600,
    "total_bytes": 2048576,
    "error": null
  }
}
```

**状态枚举:**

| 状态 | 说明 |
|------|------|
| idle | 空闲 |
| checking | 检查更新中 |
| downloading | 下载中 |
| transferring | 传输中 |
| installing | 安装中 |
| completed | 完成 |
| failed | 失败 |
