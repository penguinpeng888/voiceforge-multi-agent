# TTS（语音合成）原理详解

> 2026-04-08

---

## 一、TTS 的本质

TTS = 把**文本**转换为**语音**

核心矛盾：**离散符号**（文字）→ **连续信号**（声波）

---

## 二、传统 TTS 流水线

```
文本输入
    ↓
文本分析 → 分词、标音、韵律预测
    ↓
声学模型 → 生成声学特征（如 mel-spectrogram）
    ↓
声码器（Vocoder）→ 生成最终音频波形
```

### 各阶段功能

| 阶段 | 输入 | 输出 | 关键问题 |
|------|------|------|----------|
| 文本分析 | "你好" | 音素序列 [n i: h au] | 多音字、轻声 |
| 声学模型 | 音素 | mel频谱 | 语调、情感 |
| 声码器 | mel频谱 | 波形 | 音质、自然度 |

---

## 三、深度学习时代的 TTS

### 主流架构

```
文本 → 编码器 → 解码器 → 声码器 → 音频
```

#### 1. 编码器（Encoder）
- 把文字转换为向量表示
- 捕获语义信息

#### 2. 解码器（Decoder）
- 自回归生成声学特征
- 或非自回归（并行生成）

#### 3. 声码器（Vocoder）
- 把声学特征转为波形
- 经典：Griffin-Lim、Wavenet
- 现代：HiFi-GAN、Neural Vocoder

---

## 四、当前主流技术方案

### 方案1：两阶段 TTS
```
文本 → 声学模型 → mel频谱 → 声码器 → 音频
```
代表：Tacotron 2、Mellotron

**优点**：可解释性强
**缺点**：级联误差

### 方案2：端到端 TTS
```
文本 → 统一模型 → 音频
```
代表：VALL-E、WavLM

**优点**：消除级联误差
**缺点**：需要大量数据和计算

### 方案3：LLM + Codec
```
文本 → LLM生成Tokens → Codec解码 → 音频
```
代表：VibeVoice、CosyVoice、VoxCPM

**优点**：可以利用LLM能力
**缺点**：Tokens 设计是关键

---

## 五、语义 vs 声学

这是 TTS 的核心问题：

| 信息类型 | 内容 | 例子 |
|----------|------|------|
| **语义** | "说什么" | 文字内容、音素 |
| **声学** | "怎么说" | 语调、语速、音色、情感 |

### 理想情况
- 语义信息 → 保证内容正确
- 声学信息 → 保证自然好听

### 现实问题
- 两者纠缠在一起
- 很难完全分离

---

## 六、关键挑战

### 1. 韵律建模
- 语调（升调、降调）
- 停顿（逗号、句号）
- 重音（哪里要强调）

### 2. 长音频一致性
- 保持音色一致
- 保持情感一致
- 不出现重复或遗漏

### 3. 情感表达
- 开心、悲伤、愤怒...
- 语气（讽刺、幽默）
- 场景（新闻、故事）

### 4. 零样本克隆
- 只用短参考音频
- 复制任意说话人

---

## 七、2025-2026 最新进展

### VibeVoice（微软）
- 90分钟连续生成
- 4人对话支持
- 7.5Hz 超低帧率

### VoxCPM（MiniCPM）
- 免分词端到端
- 零样本克隆
- 情感表达

### CosyVoice（阿里）
- 流式生成
- 多语言支持
- 开源可用

---

## 八、我们的机会点

1. **轻量化**：小模型、低延迟
2. **垂直场景**：特定领域优化
3. **多模态融合**：结合视觉、文本理解
4. **边缘部署**：手机/嵌入式
