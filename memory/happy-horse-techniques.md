# Happy Horse 三大技术详解

> 2026-04-08

---

## 一、DMD-2 分布匹配蒸馏

### 背景
- 扩散模型需要数十步去噪，推理慢
- 目标：把多步模型蒸馏成少步/一步生成器

### DMD 核心思想
**分布匹配**：让一步生成器的输出分布，匹配多步扩散模型的输出分布

### 传统方法的问题
- 逐点对应：要求学生输出完全匹配老师的采样轨迹
- GAN式对抗训练：不稳定

### DMD 的创新
1. **分布级别匹配**：不要求一对一对应
2. **两个 Score Function**：
   - 评估"真实"程度的 score
   - 评估"假"程度的 score
3. **无对抗训练**：用两个 score 的差异来指导

### 训练目标

#### 1. 回归损失
```
预计算噪声-图像对
→ 学生一步生成
→ 与老师多步输出对比 (LPIPS)
```

#### 2. 分布匹配梯度
```
生成的假图像 + 随机噪声
→ 扩散模型A (预训练于真实数据)
→ 扩散模型B (持续训练于假数据)
→ 两个去噪版本的差异 = 向着更真实的方向
```

### DMD2 改进 (CVPR 2024)
- 移除回归损失要求
- 架构改进
- 只需1-5步即可生成高质量图像

### Happy Horse 的 DMD-2
- 8步去噪（不是1步，但比25-50步少）
- 无需 CFG (Classifier-Free Guidance)
- 效果依然很好

## 二、Sandwich 结构

### 核心理念
```
[首4层模态特定] → [中间32层共享] → [尾4层模态特定]
```

### 为什么这样设计？

| 层级 | 作用 | 原因 |
|------|------|------|
| 首层 | 模态特定投影 | 把不同模态转成统一 Token 序列 |
| 中间层 | 共享参数 | 学习通用知识，降低参数量 |
| 尾层 | 模态特定输出 | 把通用表示转回特定模态 |

### 与 MoT (Mixture-of-Transformers) 对比

MoT 的思路：
- 把非 embedding 参数按模态解耦
- 模态特定的 FFN、attention、layer norm
- 全局 self-attention 跨所有 token

Happy Horse 更简单：
- 只有首尾4层模态特定
- 中间32层完全共享
- 没有跨注意力

### 优势
1. **参数效率**：减少总参数量
2. **模态平衡**：避免某个模态主导
3. **简单**：比 MoT 更易实现

## 三、Per-head 门控 (Per-head Gating)

### 为什么需要门控？

> "Joint multimodal training is notoriously unstable — gradients from the audio loss can dominate or be dominated by gradients from the video loss."

多模态联合训练的问题：
- 音频梯度可能压倒视频梯度
- 或反过来
- 训练不稳定

### Happy Horse 的方案

```python
# 每个注意力头有一个学习标量门控
gate = sigmoid(weight)  # 输出 0~1 之间的值

# 门控作用在注意力头的输出上
output = output * gate
```

### 门控的作用

1. **平衡模态**：让不同模态的梯度保持平衡
2. **训练稳定**：避免某个模态主导
3. **可学习**：门控权重可以学习，自动调整

### 与其他门控机制对比

| 方法 | 说明 | 特点 |
|------|------|------|
| sigmoid 门控 | 每个头一个标量 | Happy Horse 用这个 |
| Gumbel-Softmax | 离散门控 | 可用于特征选择 |
| MoE | 专家混合 | 更复杂的路由 |

---

## 总结：三个技术的关系

```
输入 → [模态特定投影] → [共享Transformer+门控] → [模态特定输出]
                                    ↓
                              DMD-2蒸馏
                                    ↓
                              8步生成
```

1. **Sandwich**：架构设计，决定模型结构
2. **Per-head 门控**：训练技巧，让训练更稳定
3. **DMD-2**：推理优化，让生成更快

---

## 对我们的借鉴

| 技术 | 应用场景 |
|------|----------|
| Sandwich | 多模态 TTS 架构 |
| Per-head 门控 | 音频+文本联合训练 |
| DMD-2 | 把大 TTS 模型做小 |
