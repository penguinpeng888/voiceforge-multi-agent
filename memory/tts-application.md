> 2026-04-08

## 一、DMD-2 蒸馏 → TTS 加速

### 应用场景
把大 TTS 模型蒸馏成小模型，实现实时语音合成。

### 具体方案

#### 现状问题
- 神经声码器（如 WaveNet）需要几十步生成
- 自回归 TTS 速度慢
- 实时合成困难

#### DMD-2 解决方案
```
Teacher: 大模型、多步去噪（如 50 步 WaveNet）
         ↓ 分布匹配蒸馏
Student: 小模型、少步生成（如 8 步）
```

#### TTS 流水线中的位置

```
文本 → 声学模型 → mel频谱 → 神经声码器 → 音频
                              ↑
                      在这里用 DMD-2 蒸馏
```

### 预期效果
- 生成速度提升 5-10 倍
- 质量损失可接受
- 可以在消费级 GPU 上实时运行

## 二、Sandwich 结构 → TTS 架构

### 应用场景
设计一个统一处理"文本语义"和"声学特征"的 TTS 模型。

### 具体方案

```
输入：
- 文本 Token（音素/字符）
- 说话人 Embedding
- 情感/风格 Embedding

架构：
[首4层：模态特定投影]
  → 文本投影层
  → 声学投影层

[中间32层：共享 Transformer]
  → 语义-声学联合建模
  → 学习两者的交互

[尾4层：模态特定输出]
  → 输出 mel 频谱
```

### 设计细节

| 层级 | 输入 | 输出 | 作用 |
|------|------|------|------|
| 首层 | 文本 Token | 统一语义向量 | 文本编码 |
| 首层 | 说话人/风格 | 统一声学向量 | 条件编码 |
| 中间 | 语义+声学 | 融合表示 | 联合建模 |
| 尾层 | 融合表示 | mel频谱 | 声学输出 |

### 优势
- 文本和声学信息充分交互
- 共享层学习通用表示
- 端到端优化

## 三、Per-head 门控 → 多模态平衡

### 应用场景
平衡 TTS 中的多个控制因素（文本、说话人、情感、语速）。

### 具体方案

```python
# 注意力头结构
class TTSAttentionHead(nn.Module):
    def __init__(self, head_dim):
        self.gate = nn.Parameter(torch.ones(1))  # 可学习门控
    
    def forward(self, x):
        # 计算注意力
        attn_output = self.attention(x)
        # 门控作用
        output = attn_output * torch.sigmoid(self.gate)
        return output
```

### 应用场景

#### 1. 说话人 vs 文本
- 门控平衡"谁的声音"（说话人）和"说什么"（文本）
- 避免说话人特征被文本内容淹没

#### 2. 情感 vs 清晰度
- 门控平衡"情感表达"和"发音清晰度"
- 可以动态调整

#### 3. 噪声环境
- 门控可以根据噪声级别动态调整
- 噪声大时，增强清晰度门控

### 训练稳定
- 多说话人数据集训练更稳定
- 避免某个说话人主导

## 完整方案：TTS Sandwich + Per-head Gate + DMD-2

```
输入层
  ↓
[首4层：文本/说话人/情感投影 + 门控]
  ↓
[中间32层：共享Transformer + 门控]
  ↓
[尾4层：mel输出 + 门控]
  ↓
mel频谱
  ↓
[声码器（DMD-2蒸馏）]
  ↓
音频输出
```

### 预期效果
- 语义-声学联合建模
- 多因素可控
- 实时生成

---

## 简化版：先从哪个开始？

如果资源有限，建议顺序：

1. **先做 Per-head 门控**
   - 改动最小
   - 训练更稳定
   - 效果明显

2. **再做 Sandwich 结构**
   - 需要重新设计模型
   - 提升联合建模能力

3. **最后做 DMD-2 蒸馏**
   - 需要预训练大模型
   - 部署时加速
