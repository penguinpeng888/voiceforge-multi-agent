## OpenClaw 工具调用修复

### 问题
- /query 端点返回空 response.content 但 tool_calls 显示有1个调用
- MiniMax LLM 返回正确但响应内容为空

### 根因
- `_callLLM` 方法传递工具给 LLM 时，工具没有 schema 定义
- LLM 返回 response.content 为空，但 tool_calls 有值

### 修复方案
1. 添加 `_createDefaultSchema(name)` 方法预定义常见工具 schema
2. 实现两轮调用模式：
   - 第一轮：LLM 生成 tool_calls
   - 执行工具
   - 第二轮：带工具结果再次调用 LLM 获取最终响应
