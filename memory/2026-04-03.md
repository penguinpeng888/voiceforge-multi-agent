### 7. OpenClaw "做梦机制"调研（重要！）
用户问OpenClaw是否有"做梦机制"（晚上整合记忆）
- 查询了 GitHub 和官方文档
- **结论: OpenClaw 当前没有自动的夜间记忆整合机制**
- 现有机制都是被动的：
  - 只有会话结束时才会flush
  - 只有context满时才会compaction
- **但 Claude Code 确实有"后台内存整合引擎"（Background Memory Consolidation）**
  - 官方论文: https://transformer-circuits.pub/2026/emotions/index.html
  - 在用户睡觉时运行，整合记忆
- **建议**: 可为OpenClaw添加定期任务（如每天凌晨）执行：
  1. 读取本周所有 memory/YYYY-MM-DD.md
  2. 提炼重要信息更新到 MEMORY.md
  3. 删除已整合的过时日常笔记
  4. 总结学习到的教训

### 8. Claude Code 源码泄露分析
用户分享了PDF文章《从泄露代码看Claude》，核心架构：
- **自愈查询循环**: while(true) 状态机，错误恢复级联（微压缩→折叠→token升级→模型回退→最后才暴露错误）
- **工具并发批处理**: 只读工具最多10个并行，写入工具串行；延迟加载60+工具
- **后台内存整合**: 确认存在，在用户睡觉时运行
