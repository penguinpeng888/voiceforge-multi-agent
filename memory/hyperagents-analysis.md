# HyperAgents 分析 - Meta 的自改进AI代理

> 2026-04-08

---

## 核心概念

**HyperAgents** = 自指代自改进代理

把两个东西合并成一个可编辑程序：
1. **Task Agent** - 解决问题
2. **Meta Agent** - 改进 Task Agent

---

## 创新点

### 1. 解决无限回归问题
传统架构：
```
Agent → Meta-Agent → Meta-Meta-Agent → ...
```

HyperAgents：
```
Task Agent + Meta Agent = 同一个可编辑程序
```

### 2. 元认知自修改
- 不仅改进任务解决能力
- **还改进"生成改进"的机制**
- 可以修改自己的修改方法

### 3. 跨领域自改进
- 不再局限于 coding 领域
- 支持任何可计算任务

---

## 技术架构

```
DGM-H (Darwin Gödel Machine - Hyperagents)

任务执行循环：
1. Task Agent 解决问题
2. Meta Agent 生成改进方案
3. 评估改进效果
4. 应用改进到自身
5. 重复
```

---

## 核心特点

| 特点 | 说明 |
|------|------|
| 自指代 | 可以修改自己的代码 |
| 元认知 | 可以改进"改进机制"本身 |
| 开放域 | 不限于 coding |
| 累积性 | 改进可以跨领域迁移 |

---

## 实验结果

- 在多个领域随时间提升性能
- 超越没有自改进的基线
- 超越之前的自改进系统
- 改进"生成新代理"的过程（如记忆、性能跟踪）
- 跨领域迁移、累积

---

## 对我们的借鉴

1. **Task + Meta 合并**：我们的 auto-skill-evolve 可以借鉴
2. **自修改能力**：代理可以自己改进自己
3. **元认知循环**：不仅改进行为，还改进"改进方法"
