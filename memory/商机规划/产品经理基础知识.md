# 产品经理基础知识 - 程序员外包对接篇

---

## 一、基础概念

### 什么是产品经理（PM）
- 连接客户和程序员的桥梁
- 负责把客户的需求转化为可执行的技术方案
- 不需要会写代码，但要懂技术逻辑

### 外包对接中的角色
```
客户（付钱）→ 你（产品经理）→ 程序员（做事）→ 客户（验收）
```

## 三、核心流程

### 1. 需求沟通
```
客户：我想要一个网站，卖衣服的
你（提问）：
- 主要功能有哪些？（商品展示、购物车、支付、会员...）
- 有没有参考对象？（淘宝、拼多多...）
- 目标用户是谁？（年轻人？白领？...）
- 预算多少？工期有要求吗？
- 你提供设计图还是需要一起做？
```

### 2. 方案评估
```
根据客户需求，你评估：
- 技术难度：⭐⭐⭐（中等）
- 工作量：约2周
- 需要几个人：前端1人 + 后端1人
- 合理报价：3000-5000元

给客户报价：
- 基础版3000元（核心功能）
- 升级版5000元（完整功能）
- 工期：2周
```

### 3. 匹配合适的程序员
```
根据项目需求筛选：
- 需要React前端 + Node后端
- 找有商城经验的程序员
- 预算：给程序员1500-2000元
- 你赚差价：1000-1500元
```

### 4. 项目跟进
```
关键节点：
- 第1天：确认需求，签合同，收30%定金
- 第3天：出原型/设计图
- 第7天：初版完成，测试
- 第14天：修改完成，交付
- 第14+1天：收尾款

注意：
- 每天问进度
- 及时反馈客户问题
- 记录变更需求（避免扯皮）
```

### 5. 验收交付
```
验收清单：
□ 所有功能正常运行
□ 页面显示正常
□ 数据保存正确
□ 没有明显bug

交付后：
- 免费维护1个月
- 后续功能单独收费
```

## 四、必知术语

### 需求类
| 术语 | 含义 | 举例 |
|------|------|------|
| 原型 | 产品初版设计图 | 纸上的草图或Axure画的图 |
| 需求文档 | 详细功能说明 | 客户要什么、怎么实现 |
| MVP | 最小可行产品 | 先做核心功能，后面再升级 |
| 需求变更 | 中途改功能 | 加需求要加钱 |

### 技术类
| 术语 | 含义 | 举例 |
|------|------|------|
| 前端 | 用户看到界面 | 网页、App界面 |
| 后端 | 服务器逻辑 | 数据处理、业务逻辑 |
| 数据库 | 存储数据 | 用户信息、订单 |
| API | 程序之间对接 | 支付接口、地图接口 |
| UI/UX | 界面设计/用户体验 | 好看、好用 |
| 响应式 | 适配手机和电脑 | 一个网站两端都能用 |
| 适配 | 兼容不同设备 | iOS和安卓 |

### 管理类
| 术语 | 含义 | 举例 |
|------|------|------|
| 排期 | 安排开发时间 | 这周做前端，下周做后端 |
| 延期 | 没按时完成 | 要加班赶回来 |
| 验收 | 客户确认完成 | 测试没问题就交付 |
| 维护 | 交付后修改bug | 免费1个月，之后收费 |
| 迭代 | 持续升级 | 2.0版本加新功能 |

## 五、常见问题回答

### 客户问"多少钱？"
```
错误：您先说预算吧
正确：根据您刚才说的功能，我建议做基础版X元，包含XXX功能。如果您需要YYY功能，要加X元。工期约X天。
```

### 客户问"能不能便宜？"
```
错误：可以，我少赚点
正确：价格已经很低了。不过如果您能介绍客户过来，可以优惠。或者先做基础版，后面再升级。
```

### 客户要改需求怎么办？
```
错误：好的，我让程序员改
正确：改需求需要额外时间，我会让程序员评估工作量，按项目增加10-20%费用，您看可以吗？
```

### 程序员做不完怎么办？
```
错误：我也不知道怎么办
正确：先了解原因。如果是技术难点，可以加钱找更多人帮忙；如果是程序员的问题，及时换人。
```

### 客户不给钱怎么办？
```
错误：算了，不要了
正确：开始前签合同，收30%定金。交付前收30%，验收后收40%。用电子合同（有法律效力）。
```

## 六、接单话术模板

### 首次对话
```
您好！感谢咨询。

为了给您准确报价，请告诉我：
1. 您想做什么类型的项目？（网站/小程序/APP）
2. 主要功能有哪些？
3. 有没有参考对象？
4. 您的预算大概是多少？
5. 工期有要求吗？

我会根据您的需求，给出合理的方案和报价。
```

### 需求确认后
```
好的，根据您的需求，我给您整理一下：

【项目类型】：XXX
【主要功能】：1. XXX 2. XXX 3. XXX
【工期】：约X天
【报价】：XXX元（包含X次修改）

【付款方式】：
- 签约后付30%：XXX元
- 初版完成后付30%：XXX元
- 验收后付尾款40%：XXX元

【售后】：免费维护1个月

您看可以吗？如果有其他问题可以随时问我。
```

### 讨价还价
```
价格方面我们可以商量。但羊毛出在羊身上，太低价可能影响质量。

这样吧，如果您能：
- 介绍客户过来 - 优惠10%
- 一次性付清 - 优惠5%
- 做完后给好评 - 以后合作打折

您看怎么方便？
```

## 七、避坑指南

### 1. 这类客户别接
❌ 预算极低（低于市场价50%）
❌ 需求不明确（说"先做做看"）
❌ 态度傲慢（不尊重人）
❌ 之前骗过程序员（有前科）
❌ 要先交钱（可能是骗子）

### 2. 这些单别接
❌ 技术难度太高（你驾驭不了）
❌ 时间太紧（容易延期）
❌ 需求太复杂（容易扯皮）
❌ 没签合同的口头承诺

### 3. 程序员选择
✓ 好评率95%以上
✓ 有类似项目经验
✓ 回复及时（2小时内）
✓ 愿意签合同
✓ 愿意分阶段收款
