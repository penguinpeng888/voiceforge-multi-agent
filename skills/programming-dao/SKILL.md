# 第二部分：程序规划能力（最重要！）

> 这是Claude与国产模型最核心的差异——不是"怎么写"，而是"怎么想"。

## 一、规划 vs 做题

| 模型 | 思维模式 | 行为 |
|------|----------|------|
| **Claude** | 规划先行 | 先想清楚结构，再写代码 |
| **国产模型** | 做题模式 | 看到需求直接写代码 |

**本质差异**：Claude在做之前会"思考"，国产模型在接到prompt就直接"回答"。

## 二、Claude的规划流程

### 2.1 第一步：需求分析

```
输入: CLI待办事项管理器
输出:
  - 功能: add/list/done/remove
  - 数据: JSON文件存储
  - 用户: 命令行用户
  - 约束: Node.js, fs模块
```

**Claude会先列出**：功能、数据、用户、约束

### 2.2 第二步：结构设计

```
┌─────────────────────────────────────┐
│           入口 (main)                │  ← 处理命令行参数
├─────────────────────────────────────┤
│          命令处理函数                │  ← add/list/done/remove
├─────────────────────────────────────┤
│          数据读写函数                │  ← loadTodos/saveTodos
├─────────────────────────────────────┤
│           工具函数                   │  ← parseIndex/printHelp
└─────────────────────────────────────┘
```

**Claude会先画出结构图**，再开始写代码。

### 2.3 第三步：边界思考

Claude会提前想：
- 文件不存在怎么办？
- 数据损坏怎么办？
- 用户输入无效编号怎么办？
- 删除后编号变了怎么办？

### 2.4 第四步：命名规范

```
loadTodos()   ← 动词+名词，清晰表达动作
saveTodos()
addTask()     ← 动宾短语
listTasks()
```

## 二、错误处理原则

### 2.1 每个函数都要有错误处理

```javascript
// ❌ 错误示例（GLM-5）
function loadTodos() {
  const data = fs.readFileSync(DATA_FILE, 'utf-8');
  return JSON.parse(data);
}

// ✅ 正确示例（Claude）
function loadTodos() {
  try {
    if (!fs.existsSync(DATA_FILE)) return [];
    const raw = fs.readFileSync(DATA_FILE, 'utf-8');
    return JSON.parse(raw);
  } catch {
    console.error("❌ 读取数据文件失败");
    process.exit(1);  // 严重错误要退出
  }
}
```

### 2.2 错误处理层级

| 错误类型 | 处理方式 | 示例 |
|----------|----------|------|
| 可恢复 | 返回默认值 | 文件不存在返回[] |
| 严重 | process.exit(1) | 数据文件损坏 |
| 用户输入 | 友好提示+帮助 | 无效编号 |

### 2.3 错误信息要友好

```javascript
// ❌ 差
console.log("错误");

// ✅ 好
console.log(`无效的任务编号 "${numStr}"。请使用 list 查看有效编号。`);
```

## 三、参数校验原则

### 3.1 独立校验函数

```javascript
// ❌ 内联校验（GLM-5）
if (isNaN(index) || index < 1 || index > todos.length) {
  console.log('错误');
  return;
}

// ✅ 独立函数（Claude）
function parseIndex(numStr, length) {
  const num = parseInt(numStr, 10);
  if (isNaN(num) || num < 1) {
    console.error(`❌ 无效编号"${numStr}"，请输入大于0的整数。`);
    process.exit(1);
  }
  if (num > length) {
    console.error(`❌ 编号${num}超出范围，当前共有${length}项任务。`);
    process.exit(1);
  }
  return num - 1;
}
```

### 3.2 校验时机

- **越早越好**：在函数开头校验
- **统一处理**：不要在多处重复校验逻辑

### 4.1 JSDoc注释

```javascript

 * 从JSON文件加载任务列表
 * @returns {Array} 任务数组
 * @throws {Error} 文件损坏时退出

```

### 4.2 必须包含的内容

| 注释类型 | 内容 |
|----------|------|
| 函数说明 | 这个函数做什么 |
| 参数说明 | @param 类型 描述 |
| 返回值 | @returns 类型 描述 |
| 异常 | @throws 什么情况 |
